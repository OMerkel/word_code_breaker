# Internationalisation (i18n)

Word Code Breaker currently supports two languages:

| Language | Code | Keyboard layout | Special letters |
| --- | --- | --- | --- |
| English | `en` | QWERTY (A–Z) | — |
| German | `de` | QWERTZ (A–Z, Ä, Ö, Ü, ß) | Ä, Ö, Ü, ß |

The game is intentionally limited to these two languages because each language
requires a dedicated word dictionary (see [Word dictionary](#word-dictionary) below).

---

## Translation strings

All UI translations live in a single `translations` object near the top of
`javascript/html5/src/js/hmi.js`.  Each top-level key is a BCP 47 language
tag (`en`, `de`).  A missing key in a locale automatically falls back to the
English value, so you only need to supply the keys you actually want to override.

```js
const translations = {
  en: { /* … */ },
  de: { /* … */ },
};
```

#### `highscoreFormat` function signature

```js
highscoreFormat: (entry) => `${entry.attempts} tries / ${formatDuration(entry.seconds)}`
```

`entry` has the shape `{ attempts: number, seconds: number }`.
`formatDuration` is imported from `js/common.js` and formats seconds as `m:ss`.

#### `statusBody` function signature

```js
statusBody: (board) => {
  if (board.status === "won")   return "…";
  if (board.status === "lost")  return "…";
  if (board.invalidWord)        return "…";   // rejected by dictionary
  if (board.canSubmit)          return "…";   // 5 letters ready to submit
  return "…";                                  // still typing
}
```

## Word dictionary

Every supported language requires a plain-text word list file.

### File locations

| Language | Source path (repo root) | Served path (web app) |
| --- | --- | --- |
| English | `data/en_5_letters.txt` | `javascript/html5/src/data/en_5_letters.txt` |
| German | `data/de_5_letters.txt` | `javascript/html5/src/data/de_5_letters.txt` |

The web app file is a copy of the source file.
The service worker (`js/sw.js`) caches both files for offline play.

### File format

- Plain UTF-8 text, one word per line, Unix or Windows line endings.
- Each word must be **exactly 5 letters** (characters as seen after normalization).
- Words are stored lower case in the file; they are normalized to upper case at load time.
- **Exception**: `ß` must **not** be uppercased (JavaScript's `"ß".toUpperCase()` returns `"SS"`).
  The loader preserves `ß` as-is; the game stores and compares it as the single character `ß`.
- Words that do not pass the 5-character length filter after normalization are silently dropped.

### Normalization applied at load time

```text
word → trim whitespace → map each char:
         if char === 'ß'  → keep as 'ß'
         else             → char.toUpperCase()
       → keep only if length === 5
```

### Adding a new language

1. Create a word list file following the format above and place it in `data/<lang>_5_letters.txt`.
2. Copy the file to `javascript/html5/src/data/<lang>_5_letters.txt`.
3. Add the language code to `LANGUAGE_OPTIONS` in `js/common.js`:

   ```js
   export const LANGUAGE_OPTIONS = Object.freeze(["en", "de", "nl"]);
   ```

4. Add the URL mapping in `js/controller.js`:

   ```js
   const WORD_LIST_URLS = {
     en: "../data/en_5_letters.txt",
     de: "../data/de_5_letters.txt",
     nl: "../data/nl_5_letters.txt",
   };
   ```

5. Add the keyboard rows for the new language in `js/common.js`
   (model on `EN_KEYBOARD_ROWS` or `DE_KEYBOARD_ROWS`) and extend `getKeyboardRows`.

6. Add a translation block to the `translations` object in `js/hmi.js`.

7. Add a radio input for the new language to the options form in `index.html`:

   ```html
   <label><input type="radio" name="language" value="nl" />Nederlands</label>
   ```

8. Add the new word list file path to `ASSETS_TO_CACHE` in `js/sw.js` so it is
   available offline:

   ```js
   "./data/nl_5_letters.txt",
   ```
